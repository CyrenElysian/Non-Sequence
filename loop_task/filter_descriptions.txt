## Quality criteria:
1. **If the description is merely a direct restatement or negation of the loop goal (loop_step) using the same key words , delete it. Concrete observable clues that imply the goal state are acceptable.**
2. **Independent and sufficiency**: The description, **when read in the context of the given scenario and loop_step**, must allow a reasonable person to deduce the correct answer (0/1/2) without needing other descriptions from the same entry. It is worth noting that the deduction may rely on common-sense and background knowledge that the scenario implies so the description is not isolated.
3. **Unambiguous**: In the context of the scenario and loop_step, the description must point clearly to one answer.
   - If the description could genuinely be interpreted as supporting two different answers with roughly equal plausibility (a "50/50" case), remove it. (Please think carefully in context and delete cautiously.)
   - If one interpretation is clearly more natural given the context (that is, the one with greater probability), while the other is a stretch that a reasonable person would dismiss, keep the description.
   - Exercise caution: do not remove a description just because a highly unlikely alternative interpretation can be imagined.
4. **State, not explanation**: Must be a factual situation, not "because...".
5. **Correct answer**: Logical consistency with the state.
6. **Difficulty calibration** (re-evaluate and adjust if needed):
   - easy: At most one step of everyday reasoning.
   - medium: Requires a short chain or interpreting a less obvious indicator.
   - hard: Requires combining multiple clues or ruling out a plausible but false alternative. It generally contains a relatively long chain of reasoning logic.
7. **Conciseness**: The description must be concise and purely state a situation. Delete any description that contains explanatory clauses that spell out the inference, such as:
   - "confirming that..."
   - "meaning that..."
   - "which shows that..."
   - "indicating that..."
   - "this implies..."
   - "so..."
   - "therefore..."
   These clauses reduce the reasoning burden and violate the principle of "state, not explanation". If removing such a clause leaves a valid description, you may keep the remaining part; otherwise delete the entire description.
8. **For answer=2 (na)**: A description is invalid as 'na' if it contains information that presupposes the loop's state — that is, information that implies a loop is ongoing, about to stop, or has already stopped. However, such a description is NOT useless. Instead of deleting it, you must change its answer to 0 or 1 and assign an appropriate difficulty, because the description actually provides a logical basis for a loop decision.
   - If the information could only be observed after the loop ends → change answer to 0 (stop).
   - If the information could only be observed while the loop is still ongoing  → change answer to 1 (continue).

**Important**: Many high-quality descriptions describe **indirect sensory clues or phenomena ** that are, in context, highly reliable signals of the loop's state. Treat these as valid and sufficient unless they are genuinely uninformative. When in doubt, prefer keeping such descriptions over deleting them.
